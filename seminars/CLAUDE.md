# CLAUDE.md
どのセミナーでも同じ構成で記録するための、再利用プロンプト。
スタイルルールは `-repo_shin/template/template-style.md` に従うこと。


## 使い方
1. セミナーの文字起こしテキストを用意する
2. 下記プロンプトの `[セミナー名]` と `[ここに文字起こしテキストを貼る]` を書き換える
3. Claudeに貼って実行する
4. 出力結果を `seminars/YYYY-MM-DD_セミナー名.md` として保存する

**プロンプト本文**
```
以下は音声認識による文字起こしで、単語間に不自然なスペースが
入っている場合があります。まず自然な日本語文章に整形し、
その上でWhy/What/How形式に要約してください。

セミナー名：[セミナー名]

【Why（なぜ）】
- このセミナーが扱っている問題意識・背景・目的は何か

【What（なに）】
- 具体的に紹介された手段・フレームワーク・事例は何か
- 登壇者が伝えたかった核心的な内容は何か
- 1つの項目の中に複数の手段・事例・対象者などがあり複雑な場合は、
  番号付きリスト＋太字項目名＋箇条書きの形式で整理してよい

【How（どのように）】
- 実際にどう実践するか、具体的な手順や事例
- 明日から自分の仕事にどう応用できそうか

最後に「今回の学び（1〜2行）」として、自分の言葉で
要点をまとめてください。

文字起こし中に日付・登壇者が明記されていない場合は、
推測で埋めず「本文中に明記なし。ご確認ください」と記載すること。

文字起こし:
[ここに文字起こしテキストを貼る]
```


## 制約事項・禁止事項
- 文字起こしのスペース崩れ等をそのまま使わず、自然な日本語文章に整形してから要約する
- `Why`/`What`/`How`/`今回の学び`以外の見出しを追加しない
- 「今回の学び」は1〜2行に収める
- 本文中に日付・登壇者の明記がない場合、推測で埋めず「本文中に明記なし。ご確認ください」と記載する
- 見出し配下の項目が複雑・多数になる場合は、`template-style.md`の「番号付き詳細リスト」形式を使ってよい
- 出力フォーマット以外の見出しや装飾を追加しない


## 出力形式
このプロンプトをそのままClaudeに貼り付け、文字起こしを渡して要約を生成する

**出力の順番**
- `# [セミナー名]` ：セミナー名をそのままタイトルにする
- `日付` `登壇` ：分かる範囲で記載（不明な場合はご確認くださいの一文を添える）
- `## Why（なぜ）` ：問題意識・背景・目的
- `## What（なに）` ：紹介された手段・フレームワーク・事例、核心的な内容
- `## How（どのように）` ：実践方法・応用の仕方
- `## 今回の学び` ：1〜2行で自分の言葉でまとめる

**出力の例**
```markdown
# [セミナー名]

日付：YYYY-MM-DD
登壇：[登壇者名・所属]


## Why（なぜ）
（内容）


## What（なに）
（内容）


## How（どのように）
（内容）


## 今回の学び
（1〜2行）
```
