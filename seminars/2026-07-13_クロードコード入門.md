# クロードコード活用術入門

日付：仮定（本文中に明記なし。ご確認の上、ファイル名の日付を修正してください）
登壇：データサイエンス・生成AI教育サービス運営者（社会人向けリスキリング講座）


## Why（なぜ）

- 生成AIツールが乱立する中、「どのツールがどう違うのか」が分かりづらくなっている
- AI活用について「難しそう」「プログラマーでないと使えないのでは」「リスクが不安」
  という懸念を持つ社会人が多い
- 社会人はリスキリングの時間を確保しづらいが、「勉強したことをすぐ仕事に活かせる」
  という経験の蓄積は学生にはない社会人ならではの強みであり、今から学んでも遅くない
- AIを使う側と使わない側で、将来的な評価・生産性に大きな差が生まれると考えられる
  （基礎知識がないと、AIの出力が正しいか判断できず、結局仕事で使えない）


## What（なに）

**AI活用の3段階（チャット型 → Claude Cowork → Claude Code）**

1. **チャット型AI（ChatGPT・Claude等）**
   - 自然言語でやり取りし、文章作成・観点整理・構成案の提示などができる
   - ただし出力は「答えを返すだけ」で、資料への転記・体裁調整・保存などの
     実作業は人間が行う必要がある

2. **Claude Cowork（コアーク）**
   - 指示に対して作業の段取りを自分で立て、Web調査や資料参照を行い、
     Excel/PowerPoint等のファイルとして成果物を仕上げるところまで自動で行う
   - 人間は作業中に別のことをしていられ、完了通知を受けて確認するだけでよい
   - 事例：週3時間かかっていた競合調査業務が、依頼10分＋確認15分の25分に短縮
     （約80%超の時間削減）
   - 向いている人：自分一人で使う、その都度AIと相談しながら進めたい人

3. **Claude Code（コード）**
   - 作業手順を「型」として固定し、ボタン一つで毎回同じ処理を実行できる
     アプリケーションとして開発できる
   - チャットでのやり取りすら不要になり、自分以外の人にもツールとして渡せる
   - 事例：競合ウォッチャーのような、チェックボタン一つで調査を実行する
     デスクトップツールの開発
   - 向いている人：定型作業を繰り返し行う、他者にもツールとして展開したい人

**AI活用の安全性（5つの注意点）**
- AIの出力を鵜呑みにせず、人間が必ず確認する
- AIに扱わせる情報・接続範囲を最小限に決める
- 実行前に中身を確認する（重要データはバックアップを取る）
- 個人情報・機密情報をそもそもAIに渡さない設計にする
- 「仮想環境（隔離されたコンピューター内で作業させる仕組み）」を使うことで、
  ファイル誤削除や不要ツールのインストールなどのリスクを抑えられる
  - Claude Coworkは仮想環境が自動設定される
  - Claude Codeは汎用性が高い分、仮想環境構築を自分で行う必要があり、
    ここが初心者のつまずきポイントになりやすい


## How（どのように）

- まずはチャット型AIで「相談・下書き」レベルの使い方に慣れる
- 次に、自分一人で完結する定型作業（競合調査、資料作成、データ集計等）を
  Claude Coworkに任せてみる。プロンプトは「何を・どこで・どのように」を
  5W1H的に明確に伝えるだけでよく、プログラミング知識は不要
- 同じ作業を繰り返し行う、または他者にもツールとして展開したい場合に、
  Claude Codeで「道具化（アプリケーション化）」する
- 判断のフローチャート：
  「相談だけで良い→チャット型」「自分で都度確認しながら任せたい→Cowork」
  「繰り返し作業・他者展開したい→Code」
- Claude Codeを使う際は、仮想環境の構築とアクセス範囲の制御を必ず行い、
  安全性を確保してから運用する


## 今回の学び

Claude Cowork（相談しながら都度任せる）とClaude Code（型を固定して道具化する）
は「どちらが優れているか」ではなく「定型化の度合いと展開範囲」で使い分けるもの、
という整理が実務的だった。特に「仮想環境なしでClaude Codeを使うのはリスクが高い」
という点は、今後Claude Codeを本格的に使う上で最初に押さえておくべき注意点。
