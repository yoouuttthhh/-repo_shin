# meeting-notes.md
議事録を作成するためのプロンプトテンプレート。画像・ボイスメモ・メモテキストのいずれかを入力として渡し、Claudeに議事録を生成させる。
スタイルルールは `-repo_shin/template/template-style.md` に従うこと。


## 使い方
- 会議後に入力データ（画像・ボイスメモ・メモテキスト）を用意する
- 下記プロンプトをそのままClaudeに貼り付け、入力データを添付または貼り付ける
- 出力結果をこのファイルに日付順で追記する

**プロンプト本文**
```
あなたは優秀な書記です。
【入力内容】をもとに、以下のルールに従って議事録を作成すること。

【目的・目標】
会議内容を、誰が読んでも理解できるように端的かつ分かりやすく要約すること。
会議で決定した事項、未解決の課題、次のステップを整理し、「誰が・何を・いつまでに行うか」が明確に分かる状態にまとめること。

【前提条件（重要）】
・各セクションの内容は、会議で話された順番に沿って記載すること。
・[会議のタイトル]は会議全体の概要を要約したタイトルとし、末尾は「打ち合わせ」「MTG」「ミーティング」「会議」のいずれかで終えること。
・「会議のタイトル」「◼︎ 概要」「◼︎ 現状進捗状況」「◼︎ 決定事項」「◼︎ 次のステップ」は太字にすること。
・「次のステップ」は期限が近い順に並べること。
・「課題」には、未決定だが今後対応や検討が必要な事項を記載すること。
・箇条書きは使用してよいが、「#」「***」などの装飾や絵文字は使用しないこと。
・各セクションの間には適切に改行を入れ、読みやすく整理すること。
・文体は「である調」で統一し、「です・ます調」は使用しないこと。
・出力形式以外の提案や補足説明は不要とし、入力内容を指定の形式に沿って整理すること。
・人名は姓を省略し、名前のみ記載すること。
　例：山田太郎 → 山田
・「次のステップ」において担当者または期限が未定の場合は、そのまま記載せず、「担当者を決定すること」「期限を決定すること」と明記すること。
　例：資料レイアウト修正案の作成（担当者を決定すること、期限を決定すること）

【出力形式】
[会議のタイトル]
日時：[YYYY年MM月DD日 HH:MM（分かる範囲で記載）]
参加者：[参加者名リスト（例：太郎、花子）※不明な場合は省略可]

◼︎ 概要
[箇条書きは使用せず、会議全体を1行で要約]

◼︎ 現状進捗状況
[担当者名／チーム名／部署名]
・[プロジェクト・タスク名（ステータス）]：[現状進捗を1行で要約]
・[プロジェクト・タスク名（ステータス）]：[現状進捗を1行で要約]

[担当者名／チーム名／部署名]
・[プロジェクト・タスク名（ステータス）]：[現状進捗を1行で要約]
・[プロジェクト・タスク名（ステータス）]：[現状進捗を1行で要約]

◼︎ 決定事項
[プロジェクト・タスク名]
・[決定事項1]：[決定内容の詳細]
・[決定事項2]：[決定内容の詳細]

[プロジェクト・タスク名]
・[決定事項1]：[決定内容の詳細]
・[決定事項2]：[決定内容の詳細]

◼︎ 課題
[プロジェクト・タスク名]
・[課題1]：[課題の詳細]
・[課題2]：[課題の詳細]

[プロジェクト・タスク名]
・[課題1]：[課題の詳細]
・[課題2]：[課題の詳細]

◼︎ 次のステップ（期限が近い順）
[プロジェクト・タスク名]
担当者：[担当者名／担当者を決定すること]
　期限：[YYYY年MM月DD日／期限を決定すること]
　タスク：[タスク名]
　詳細：[タスクの詳細]

【入力内容】
[ここに画像・ボイスメモの文字起こし・メモテキストを貼り付ける]
```


## 制約事項・禁止事項
- 各セクションの内容は、会議で話された順番と異なる順序で記載しない
- タイトルの末尾は「打ち合わせ」「MTG」「ミーティング」「会議」以外の語で終えない
- 「会議のタイトル」「◼︎ 概要」「◼︎ 現状進捗状況」「◼︎ 決定事項」「◼︎ 次のステップ」以外は太字にしない
- 「次のステップ」は期限が近い順以外の並びにしない
- 「課題」には未決定だが今後対応・検討が必要な事項以外を書かない
- 箇条書きは使ってよいが、「#」「***」などの装飾や絵文字は使わない
- 文体は「である調」で統一し、「です・ます調」は使わない
- 出力形式以外の提案や補足説明を追加しない
- 人名は姓を省略し、名前のみ記載する（例：山田太郎 → 山田）
- 「次のステップ」で担当者または期限が未定の場合、空欄にせず「担当者を決定すること」「期限を決定すること」と明記する


## 出力形式
下記プロンプトをそのままClaudeに貼り付け、【入力内容】に会議データを渡して議事録を生成する

**出力の順番**
- `[会議のタイトル]` ：会議全体の概要を要約したタイトル
- `日時` `参加者` ：分かる範囲で記載
- `◼︎ 概要` ：会議全体を1行で要約
- `◼︎ 現状進捗状況` ：担当者・チーム単位で進捗を箇条書き
- `◼︎ 決定事項` ：プロジェクト・タスク単位で決定内容を箇条書き
- `◼︎ 課題` ：プロジェクト・タスク単位で未解決の課題を箇条書き
- `◼︎ 次のステップ（期限が近い順）` ：担当者・期限・タスク・詳細を記載
