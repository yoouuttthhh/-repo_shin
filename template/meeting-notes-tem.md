# meeting-notes.md
議事録を作成するためのプロンプトテンプレート。画像・ボイスメモ・メモテキストのいずれかを入力として渡し、Claudeに議事録を生成させ、新しい日付順に並べさせる。
スタイルルールは `-repo_shin/template/template-style.md` に従うこと。


## 使い方
- 会議後に入力データ（画像・ボイスメモ・メモテキスト）を用意する
- 下記プロンプトをそのままClaudeに貼り付け、入力データを添付または貼り付ける
- 出力結果を`meeting-notes.md`の先頭（最新が上）に`## [YYYY-MM-DD]　会議のタイトル`単位で追記する

**プロンプト本文**
```
あなたは優秀な書記です。
【入力内容】をもとに、以下のルールに従って議事録を作成すること。

【目的・目標】
会議内容を、誰が読んでも理解できるように端的かつ分かりやすく要約すること。
会議で決定した事項、未解決の課題、次のステップを整理し、「誰が・何を・いつまでに行うか」が明確に分かる状態にまとめること。

【前提条件（重要）】
・各セクションの内容は、会議で話された順番に沿って記載すること。
・[会議のタイトル]は会議全体の概要を要約したタイトルとし、末尾は「打ち合わせ」「MTG」「ミーティング」「会議」のいずれかで終えること。
・見出し`## [YYYY-MM-DD]　会議のタイトル`と各項目のラベル（`**ラベル**`）以外は太字にしないこと。
・「次のステップ」は期限が近い順に並べること。
・「課題」には、未決定だが今後対応や検討が必要な事項を記載すること。
・各項目は箇条書き（`- `）にしプレビュー表示時に改行されるようにすること。「・」などの記号を先頭につけただけの箇条書きは使用しないこと。
・各セクションの間には適切に改行を入れ、読みやすく整理すること。
・文体は「である調」で統一し、「です・ます調」は使用しないこと。
・出力形式以外の提案や補足説明は不要とし、入力内容を指定の形式に沿って整理すること。
・人名は姓を省略し、名前のみ記載すること。
　例：山田太郎 → 山田
・「次のステップ」において担当者または期限が未定の場合は、そのまま記載せず、「担当者未定」「期限未定」と明記すること。
　例：資料レイアウト修正案の作成：詳細（担当者未定、期限未定）
・新しい会議は`meeting-notes.md`の先頭に追記し、常に新しい日付順（逆時系列）に並べること。

【出力形式】
## [YYYY-MM-DD]　[会議のタイトル]
[箇条書きは使用せず、会議全体を1行で要約]
- **日時**：[YYYY年MM月DD日 HH:MM（分かる範囲で記載）]
- **参加者**：[参加者名リスト（例：太郎、花子）※不明な場合は省略可]

### 現状進捗状況
- **[プロジェクト・タスク名]**：[現状進捗を1行で要約]（[担当者]、[ステータス]）
- **[プロジェクト・タスク名]**：[現状進捗を1行で要約]（[担当者]、[ステータス]）

### 決定事項
- **[プロジェクト・タスク名]**：[決定内容の詳細]：[決定理由]
- **[プロジェクト・タスク名]**：[決定内容の詳細]：[決定理由]

### 課題
- **[プロジェクト・タスク名]**：[課題の詳細]（[背景・理由]）
- **[プロジェクト・タスク名]**：[課題の詳細]（[背景・理由]）

### 次のステップ（期限が近い順）
- **[タスク名]**：[詳細]（[担当者／担当者未定]、[YYYY年MM月DD日／期限未定]まで）
- **[タスク名]**：[詳細]（[担当者／担当者未定]、[YYYY年MM月DD日／期限未定]まで）

【入力内容】
[ここに画像・ボイスメモの文字起こし・メモテキストを貼り付ける]
```


## 制約事項・禁止事項
- 各セクションの内容は、会議で話された順番と異なる順序で記載しない
- 会議タイトルの末尾は「打ち合わせ」「MTG」「ミーティング」「会議」以外の語で終えない
- 見出し（`## [YYYY-MM-DD]　会議のタイトル`）と項目ラベル（`**ラベル**`）以外は太字にしない
- 「次のステップ」は期限が近い順以外の並びにしない
- 「課題」には未決定だが今後対応・検討が必要な事項以外を書かない
- 各項目は箇条書き（`- `）で記載し、「・」などの記号を先頭につけただけの箇条書きにしない（プレビュー表示時に改行されないため）
- 「#」「***」などの装飾や絵文字は使わない
- 文体は「である調」で統一し、「です・ます調」は使わない
- 出力形式以外の提案や補足説明を追加しない
- 人名は姓を省略し、名前のみ記載する（例：山田太郎 → 山田）
- 「次のステップ」で担当者または期限が未定の場合、空欄にせず「担当者未定」「期限未定」と明記する
- 新しい会議が常にファイルの先頭にくるようにする（逆時系列）


## 出力形式
下記プロンプトをそのままClaudeに貼り付け、【入力内容】に会議データを渡して議事録を生成する

**出力の順番**
- `## [YYYY-MM-DD]　会議のタイトル` ：日付と、会議全体の概要を要約したタイトルを見出しにする
- 概要 ：見出し直下にラベルなしで会議全体を1行で要約
- `**日時**` `**参加者**` ：分かる範囲で記載し、箇条書きにする
- `### 現状進捗状況` ：プロジェクト・タスク単位で、担当者とステータスを添えて箇条書きにする
- `### 決定事項` ：プロジェクト・タスク単位で、決定内容と理由を箇条書きにする
- `### 課題` ：プロジェクト・タスク単位で、未解決の課題と背景を箇条書きにする
- `### 次のステップ（期限が近い順）` ：タスク単位で、詳細・担当者・期限を箇条書きにする
