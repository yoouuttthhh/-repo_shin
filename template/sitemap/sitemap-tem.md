# sitemap-tem.md
サイトのページ構成変更・新規追加・修正・削除に対応し、`01_Plan/Sitemap/` 内のsitemap.md・CSVを新規作成・更新・削除するためのプロンプトテンプレート。
スタイルルールは `template/template-style.md` に従うこと。


## 使い方
- サイトのページ構成変更・新規追加・修正・削除が発生した際、既存の `01_Plan/Sitemap/sitemap.md`・`sitemap.csv` を新規作成・更新・削除する。
- 先方からサイトマップデータ（Excel・PDF・メール文面など）をもらえる場合が多いため、そのデータをもとに整理する。
- プロジェクト開始時に本テンプレートで新規作成した場合も、出力の構成・ルールは同じものを使う。
- ページの追加・削除・名称変更・構成変更は必ずこのファイル・CSVを同時に更新する
- 変更内容は都度「更新履歴」に新しいバージョン（ver.X.X）として追記する（新しい履歴ほど上に記載する逆時系列）
- 変更後は[CLAUDE.md](./CLAUDE.md)または[README.md](./README.md)のファイルディレクトリも確認・更新する


## 背景・前提条件
プロジェクト開始時にゼロからサイトマップを調査する場合は [sitemap-research.md](./sitemap-research.md) を使用し、出力仕様は本ファイルに従う。
スタイルルールは `-repo_shin/Template/template-style.md` に従うこと。

**バージョンの付け方（sitemap.md・CSV共通）**
- 初版作成時は `ver.0.1` とする
- 通常の追加・修正・削除は `0.1` ずつ増やす（例：ver.0.1 → ver.0.2 → ver.0.3）
- 大規模なページ構成の刷新（リニューアル等）の場合は整数を繰り上げる（例：ver.0.9 → ver.1.0）
- バージョン表記は `ver.X.X（YYYY-MM-DD）` で統一する



## 制約事項・禁止事項
- ページの追加・削除・名称変更・構成変更が起きたら、必ず `01_Plan/Sitemap/` 内のsitemap.md・CSVの2形式を同時に更新する
- レポジトリーの中にフォルダーがない場合`Sitemap/`のフォルダーを作成し、sitemap.md・CSVの2形式を格納する
- 変更内容はsitemap.md・CSVそれぞれの「更新履歴」にバージョン（ver.X.X）として記録する
- 変更後はCHANGELOG.mdやCLAUDE.mdのファイルディレクトリも確認・更新する
- CLAUDE.mdのコンテキストセクションに「ページ構成の確認は `01_Plan/Sitemap/sitemap.md` を参照」と明記しておく
- 備考に明記するという指示がなければ、備考は空欄にする
- 404や非公開ページなど除外ページがなければ、省略する
- グローバルナビゲーションをベースに階層を整理する
- ディレクトリーは必ず ``をつける
- リンクがわかるリンク先にはリンク処理をする（例：[製品パンフレット(623KB)（PDF）](URL)）
- 追加・修正・削除はAdded・Changed・Removedに理由を書いて、ページ一覧の備考には理由は書かない
- ページ一覧のページ名には`（）`をつけない　（除外ページ名も同様）
- Added・Changed・Removedに追加・修正・削除を記録する際には`- **ページ名**：理由`のスタイルを作成すること 
- ページ構成にパスは書かない
- 除外ページには404・非公開・検証用と判断し一覧から除外したもの以外にもnoindex対応する、Removed・削除するものを含む
- ページ一覧からRemovedしたものは削除し除外ページに記録

## 出力形式
このルールに従って `01_Plan/Sitemap/sitemap.md`・`sitemap.csv` を作成・更新する。

**出力の順番（sitemap.md）**
- `# sitemap.md` ：ファイル名をそのまま記載
- 概要・関連ファイル（CSVへのリンク）を記載
- `## [YYYY-MM-DD]` ：リリースした日付ごとにセクションを分ける（新しい日付が上）
- `### Added` ：バージョンごとの変更内容を記載
- `### Changed` ：バージョンごとの変更内容を記載
- `### Removed` ：バージョンごとの変更内容を記載
- `## ページ構成` ：WBS形式のツリーで階層を記載
- `## ページ一覧` ：階層・ページ名・パス・URL・メモを表形式で記載
- `**除外ページ**` ：404・非公開・検証用と判断し一覧から除外

**sitemap.mdの例**
```markdown

[サイト名](url)のページ構成（サイトマップ）。
Claudeはこのファイルを参照してページ構成を把握し、
実装・デザイン・コンテンツ作成の軸として使用すること。

関連ファイル：[sitemap.csv](./sitemap.csv)

**CSVの例**
- 第1〜3階層を列で分けた構成。列はA〜L（12列）
- 更新のたびに既存データは残したまま、末尾に空白2行を挟んで新しいバージョンブロックを追加する（上書きしない）
- 各バージョンブロックの先頭にはバージョン名の行を1行入れる
- **現在の最新状態は、常にファイル末尾（一番下）のバージョンブロック**
- 新しいバージョンブロックの備考列には、sitemap.md（ページ一覧）と同様に変更があった項目へ `Added`・`Changed`・`Removed` を記載する（理由は書かず、タグのみ。理由はsitemap.md側のAdded/Changed/Removedに書く）
- 過去のバージョンブロックにはタグを付けず、その時点の内容をそのまま残す

```
