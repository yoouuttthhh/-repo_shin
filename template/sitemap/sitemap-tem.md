# sitemap-tem.md
サイトのURLからページ構成（サイトマップ）を取得し、Markdown階層・CSV・XLSXの3形式で出力するためのプロンプトテンプレート。
スタイルルールは `-repo_shin/template/template-style.md` に従うこと。


## 使い方
Claudeに作るサイトのページ構成を正確に伝え、実装・デザイン・コンテンツ作成の軸が崩れないようにするためのファイル `01_Plan/sitemap.md` を、以下2種類のプロンプトのいずれかで作成・更新する。


### プロンプト
稼働中サイトのURLから実際のサイトマップを取得・整理する時に使う。

1. 「背景・前提条件」の方法でサイトマップデータを用意する
2. 下記プロンプトをClaudeに貼り付ける
3. 【入力データ】にURLまたはコピーしたデータを貼り付ける
4. 出力形式を指定して実行する

```
以下のサイトのページ構成（サイトマップ）を取得・整理してください。

【サイト情報】
サイト名：[例：双日ケミカルトレーディング]
URL：[例：https://www.chemical-trading.sojitz.com/]

【出力形式】
以下から選んで指定してください（複数選択可）。
- Markdown階層形式
- CSV（sitemap.csv）
- XLSX（sitemap.xlsx）

【ルール】
- ページ名・URL・階層（1〜3階層）・備考（推定かどうか）を記載する
- 実際に確認できたページは「確認済み」、推定のページは「推定」と備考に明記する
- 404や非公開ページは含めない
- グローバルナビゲーションをベースに階層を整理する

【入力データ】
[以下のいずれかを貼り付ける]
・URLのみ：https://...
・sitemap.xmlの内容：（コピペ）
・sitemap.htmlの内容：（コピペ）
・ナビゲーションのHTML：（コピペ）
```

### サイトマップの精度を上げる方法（重要）
精度は入力方法によって大きく変わる。以下の順で精度が高い。

| 方法 | 精度 | 手順 |
|---|---|---|
| **①sitemap.htmlをコピペ** | ◎ 最高 | ブラウザで `サイトURL/sitemap.html` を開き全文コピー |
| **②sitemap.xmlをコピペ** | ◎ 最高 | ブラウザで `サイトURL/sitemap.xml` を開き全文コピー |
| **③ナビゲーションHTMLをコピペ** | ○ 高い | ブラウザの検証ツール（F12）でヘッダーのHTMLをコピー |
| **④URLのみ渡す** | △ 部分的 | Claudeが検索インデックスから推定。非公開ページは取れない |

まず `サイトURL/sitemap.xml` または `サイトURL/sitemap.html` をブラウザで開いてみる。
開けた場合はその内容をコピーして貼り付けると最も精度が高くなる。



## 背景・前提条件

### サイトマップとは
サイトマップ（構成図）とは、Webサイト全体の構成を図にまとめたもの。
各Webページに階層を設けてコンテンツごとに分類する。
サイトマップはWebサイトの地図であり、見るだけでWebページの場所・概要がわかる。
- **Webサイトの成功につながる**：集客・ブランディング・信頼関係の構築につながる。ディレクトリ構造はSEO対策の重要要素の一つで、構造が乱雑だと検索エンジンが正しく評価できずSEO効果が落ちる。ユーザー目線でも構造が乱雑だと使い勝手が悪く、目的の情報が見つけづらい
- **制作会社に依頼しやすくなる**：何を依頼すべきかが具体的に把握でき、見積もりや自社の予算が立てやすくなる。また制作会社が進めている内容をリアルタイムで把握できる
- **複数のスタッフで共有できる**：サイト構造を複数のスタッフで共有して管理業務の効率化が可能。たとえばアクセス解析や検索流入の解析で弱いカテゴリーを分析し、スタッフ同士で共有するのが容易になる

| | 役割 |
|---|---|
| サイトマップ | Webサイト全体の構成図 |
| ワイヤーフレーム | Webページ単位の構成図（配置要素・レイアウトを定めるもの） |

サイトマップを用意したあとに、必要なワイヤーフレームのWebページを決定する。


## 制約事項・禁止事項
- ページの追加・削除・名称変更が起きたら、必ず `01_Plan/sitemap.md` を更新する
- ページ構成が変わった場合は `01_Plan/CHANGELOG.md` にも記録する
- 変更後はCLAUDE.mdのファイルディレクトリも確認・更新する
- CLAUDE.mdのコンテキストセクションに「ページ構成の確認は `01_Plan/sitemap.md` を参照」と明記しておく
- 実際に確認できたページは「確認済み」、推定のページは「推定」と備考に明記する
- 404や非公開ページは含めない
- グローバルナビゲーションをベースに階層を整理する


## 出力形式
このルールに従って以下の構成で `01_Plan/sitemap.md` を作成する

**出力の順番**
- `# sitemap.md` ：ファイル名をそのまま記載
- 概要と最終更新日を2行以内で記載
- `## ページ構成` ：WBS形式のツリーで階層を記載
- `## ページ一覧` ：階層・ページ名・パス・URL・メモを表形式で記載
- `## 更新ルール` ：このファイルの更新ルールを箇条書きで記載

**出力の例**
```markdown
# sitemap.md
[サイト名]のページ構成（サイトマップ）。
Claudeはこのファイルを参照してページ構成を把握し、
実装・デザイン・コンテンツ作成の軸として使用すること。

最終更新：YYYY-MM-DD


## ページ構成

[サイト名]/
├─ TOP /
├─ [第1階層ページ名] /[パス]/
│  ├─ [第2階層ページ名] /[パス]/
│  └─ [第2階層ページ名] /[パス]/
└─ [第1階層ページ名] /[パス]/


## ページ一覧

| 階層 | ページ名 | パス | URL | メモ |
|---|---|---|---|---|
| 1 | TOP | / | https://example.com/ | |
| 2 | [ページ名] | /[パス]/ | https://example.com/[パス]/ | |
| 3 | [ページ名] | /[パス]/ | https://example.com/[パス]/ | |


## 更新ルール
- ページの追加・削除・名称変更は必ずこのファイルを更新する
- 変更内容は `01_Plan/CHANGELOG.md` に記録する
- 変更後はCLAUDE.mdのファイルディレクトリも確認・更新する
```


### CSV（sitemap.csv）の例
第1〜3階層を列で分けた構成。列はA〜L（12列）。

```
（空白行）
,,第1階層,,第2階層,,第3階層,,階層,,URL,メモ
（空白行）
,1,TOP,,,,,,/,,https://example.com/,
,,,1,会社について,,,,/company,,https://example.com/company,
,,,,,1,会社概要,,/company/about,,https://example.com/company/about,
（最終行）合計,,合計,,合計,ページ合計,総合計,,,
```

### XLSX（sitemap.xlsx）の仕様
- 列構成はsitemap/フォルダーにあるsitema-sample.xlsxとsitemap-sample.csvと同じ（A〜L・12列）
- ヘッダー行・データ行ともに背景色なし
- 枠線なし
- 最終行に階層別ページ数の合計式（SUM関数）
- フォント：Arial統一
