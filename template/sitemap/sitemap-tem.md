# sitemap-tem.md
サイトのページ構成変更・新規追加・修正・削除に対応し、`01_Plan/Sitemap/` 内のsitemap.md・CSVを新規作成・更新・削除するためのプロンプトテンプレート。
プロジェクト開始時にゼロからサイトマップを調査する場合は [sitemap-research.md](./sitemap-research.md) を使用し、出力仕様は本ファイルに従う。
スタイルルールは `-repo_shin/Template/template-style.md` に従うこと。


## 使い方
サイトのページ構成変更・新規追加・修正・削除が発生した際、既存の `01_Plan/Sitemap/sitemap.md`・`sitemap.csv` を新規作成・更新・削除する。先方からサイトマップデータ（Excel・PDF・メール文面など）をもらえる場合が多いため、そのデータをもとに整理する。プロジェクト開始時に本テンプレートで新規作成した場合も、出力の構成・ルールは同じものを使う。


### プロンプト
1. 先方からの改修依頼内容・サイトマップデータを用意する
2. 下記プロンプトをClaudeに貼り付ける
3. 【変更内容】に依頼内容を貼り付ける
4. 出力形式を指定して実行する

```
以下の変更内容をもとに、01_Plan/Sitemap/ 内のsitemap.md・sitemap.csvを更新してください。

【サイト情報】
サイト名：[例：双日ケミカルトレーディング]

【変更内容】
[追加・修正・削除するページの情報を貼り付ける。先方から届いたExcel/PDF/メール文面などをそのまま貼ってよい]

【出力形式】
既存の01_Plan/Sitemap/内のファイルを直接更新する（新規ファイルは作らない）。
- sitemap.md（Markdown階層＋ページ一覧＋更新履歴）
- sitemap.csv（新しいバージョンブロックを追加）

【ルール】
- 変更差分は「更新履歴」にバージョン（ver.X.X）として記録する
- ページ名・URL・階層（1〜3階層）・備考を記載する
- 実際に確認できたページは「確認済み」、推定のページは「推定」と備考に明記する
- 404や非公開ページは含めない
```


## 背景・前提条件

### サイトマップとは
サイトマップ（構成図）とは、Webサイト全体の構成を図にまとめたもの。
各Webページに階層を設けてコンテンツごとに分類する。
サイトマップはWebサイトの地図であり、見るだけでWebページの場所・概要がわかる。
- **Webサイトの成功につながる**：集客・ブランディング・信頼関係の構築につながる。ディレクトリ構造はSEO対策の重要要素の一つで、構造が乱雑だと検索エンジンが正しく評価できずSEO効果が落ちる。ユーザー目線でも構造が乱雑だと使い勝手が悪く、目的の情報が見つけづらい
- **制作会社に依頼しやすくなる**：何を依頼すべきかが具体的に把握でき、見積もりや自社の予算が立てやすくなる。また制作会社が進めている内容をリアルタイムで把握できる
- **複数のスタッフで共有できる**：サイト構造を複数のスタッフで共有して管理業務の効率化が可能。たとえばアクセス解析や検索流入の解析で弱いカテゴリーを分析し、スタッフ同士で共有するのが容易になる

| | 役割 |
|---|---|
| サイトマップ | Webサイト全体の構成図 |
| ワイヤーフレーム | Webページ単位の構成図（配置要素・レイアウトを定めるもの） |

サイトマップを用意したあとに、必要なワイヤーフレームのWebページを決定する。


## 制約事項・禁止事項
- ページの追加・削除・名称変更・構成変更が起きたら、必ず `01_Plan/Sitemap/` 内のsitemap.md・CSVの2形式を同時に更新する
- 変更内容はsitemap.md・CSVそれぞれの「更新履歴」にバージョン（ver.X.X）として記録する（履歴はファイル自体に残すため、別途CHANGELOG.mdへの転記は不要）
- 変更後はCLAUDE.mdのファイルディレクトリも確認・更新する
- CLAUDE.mdのコンテキストセクションに「ページ構成の確認は `01_Plan/Sitemap/sitemap.md` を参照」と明記しておく
- 実際に確認できたページは「確認済み」、推定のページは「推定」と備考に明記する
- 404や非公開ページは含めない
- グローバルナビゲーションをベースに階層を整理する


## 出力形式
このルールに従って `01_Plan/Sitemap/sitemap.md`・`sitemap.csv` を作成・更新する。

**出力の順番（sitemap.md）**
- `# sitemap.md` ：ファイル名をそのまま記載
- 概要・最終更新日・関連ファイル（CSVへのリンク）を記載
- `## ページ構成` ：WBS形式のツリーで階層を記載
- `## ページ一覧` ：階層・ページ名・パス・URL・メモを表形式で記載
- `## 更新履歴` ：バージョンごとの変更内容を記載
- `## 更新ルール` ：このファイルの更新ルールを箇条書きで記載

**sitemap.mdの例**
```markdown
# sitemap.md
[サイト名]のページ構成（サイトマップ）。
Claudeはこのファイルを参照してページ構成を把握し、
実装・デザイン・コンテンツ作成の軸として使用すること。

最終更新：YYYY-MM-DD
関連ファイル：[sitemap.csv](./sitemap.csv)

- 変更内容は都度「更新履歴」に新しいバージョン（ver.X.X）として追記する（新しい履歴ほど上に記載する逆時系列）
- 変更後はCLAUDE.mdのファイルディレクトリも確認・更新する
```


### バージョンの付け方（sitemap.md・CSV共通）
- 初版作成時は `ver.0.1` とする
- 通常の追加・修正・削除は `0.1` ずつ増やす（例：ver.0.1 → ver.0.2 → ver.0.3）
- 大規模なページ構成の刷新（リニューアル等）の場合は整数を繰り上げる（例：ver.0.9 → ver.1.0）
- バージョン表記は `ver.X.X（YYYY-MM-DD）` で統一する


### CSVの例
- 第1〜3階層を列で分けた構成。列はA〜L（12列）
- 更新のたびに既存データは残したまま、末尾に空白2行を挟んで新しいバージョンブロックを追加する（上書きしない）
- 各バージョンブロックの先頭にはバージョン名の行を1行入れる
- **現在の最新状態は、常にファイル末尾（一番下）のバージョンブロック**
