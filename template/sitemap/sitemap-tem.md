# sitemap-tem.md
サイトのURLからページ構成（サイトマップ）を取得し、Markdown階層・CSV・XLSXの3形式で出力するためのプロンプトテンプレート。
スタイルルールは `-repo_shin/template/template-style.md` に従うこと。


## サイトマップとは

サイトマップ（構成図）とは、Webサイト全体の構成を図にまとめたもの。
各Webページに階層を設けてコンテンツごとに分類する。
サイトマップはWebサイトの地図であり、見るだけでWebページの場所・概要がわかる。

**ワイヤーフレームとの違い**

| | 役割 |
|---|---|
| サイトマップ | Webサイト全体の構成図 |
| ワイヤーフレーム | Webページ単位の構成図（配置要素・レイアウトを定めるもの） |

サイトマップを用意したあとに、必要なワイヤーフレームのWebページを決定する。


## sitemap.mdの目的

Claudeに作るサイトのページ構成を正確に伝え、実装・デザイン・コンテンツ作成の軸が崩れないようにするためのファイル。
Claude Codeがこのファイルを参照することで、どのページが存在するか・どの階層にあるかを常に把握した状態で作業できる。

- ページの追加・削除・名称変更が起きたら、必ずこのファイルも更新する
- CLAUDE.mdのコンテキストセクションに「ページ構成の確認は `01_Plan/sitemap.md` を参照」と明記しておくと効果的


## sitemap.md生成プロンプト

新しいプロジェクトでsitemap.mdを作成する時に使う。

```
sitemap-tem.mdを参照して、以下の情報をもとに
`01_Plan/sitemap.md` を新規作成してください。

【サイト情報】
サイト名：[サイト名を入力]
URL：[本番URLまたは予定URLを入力]
目的：[例：企業サイトのリニューアル。採用・事業紹介・お問い合わせが主な目的]

【ページ構成のイメージ】
[わかっている範囲でページ名・階層を箇条書きで入力。
 不明な場合は「目的から提案して」と書くだけでOK]

【出力形式】
- Markdown階層形式（必須）
- CSV（sitemap.csv）
- XLSX（sitemap.xlsx）
[不要な形式は削除してください]

【Claudeへの注意事項】
- このsitemap.mdは実装・デザイン・コンテンツ作成の軸になるファイルです
- ページを追加・削除・変更する場合は必ずこのファイルも同時に更新してください
- ページ構成が変わった場合はCHANGELOG.mdにも記録してください
```

## なぜサイトマップが必要か

**Webサイトの成功につながる**
サイトマップ（構成図）を作成すれば、Webサイトを通じた集客・ブランディング・信頼関係の構築などにつながる。
Webサイトの設計（ディレクトリ構造）はSEO対策において重要な要素の一つ。
サイト構造が乱雑だと検索エンジンがWebページを正しく評価できず、SEOの効果が落ちる可能性がある。
また、ユーザー目線でもサイト構造が乱雑な状態は使い勝手が悪く、目的の情報が見つけづらい。
サイトマップを適切に作成すればWebサイトの構造を整理できるため、SEO効果・ユーザー体験の向上が期待できる。

**制作会社に依頼しやすくなる**
サイトマップ（構成図）があれば、制作会社に何を依頼すべきかが具体的に把握できる。
「制作会社が見積もりを出しやすくなる」「自社の予算が立てやすくなる」といったメリットがある。
また、サイトマップがあれば制作会社が進めている内容をリアルタイムで把握できる点も大きなメリット。

**Webサイトの管理を複数のスタッフで共有できる**
サイトマップ（構成図）があれば、サイト構造を複数のスタッフで共有して管理業務の効率化が可能。
たとえば、アクセス解析や検索流入の解析で弱いカテゴリーを分析し、スタッフ同士で共有するのが容易になる。


## 精度を上げる方法（重要）

精度は入力方法によって大きく変わる。以下の順で精度が高い。

| 方法 | 精度 | 手順 |
|---|---|---|
| **①sitemap.htmlをコピペ** | ◎ 最高 | ブラウザで `サイトURL/sitemap.html` を開き全文コピー |
| **②sitemap.xmlをコピペ** | ◎ 最高 | ブラウザで `サイトURL/sitemap.xml` を開き全文コピー |
| **③ナビゲーションHTMLをコピペ** | ○ 高い | ブラウザの検証ツール（F12）でヘッダーのHTMLをコピー |
| **④URLのみ渡す** | △ 部分的 | Claudeが検索インデックスから推定。非公開ページは取れない |

**まず `サイトURL/sitemap.xml` または `サイトURL/sitemap.html` をブラウザで開いてみる。**
開けた場合はその内容をコピーして貼り付けると最も精度が高くなる。


## 使い方

1. 上記の方法でサイトマップデータを用意する
2. 下記プロンプトをClaudeに貼り付ける
3. 【入力データ】にURLまたはコピーしたデータを貼り付ける
4. 出力形式を指定して実行する


## プロンプト本文

```
以下のサイトのページ構成（サイトマップ）を取得・整理してください。

【サイト情報】
サイト名：[例：双日ケミカルトレーディング]
URL：[例：https://www.chemical-trading.sojitz.com/]

【出力形式】
以下から選んで指定してください（複数選択可）。
- Markdown階層形式
- CSV（sitemap.csv）
- XLSX（sitemap.xlsx）

【ルール】
- ページ名・URL・階層（1〜3階層）・備考（推定かどうか）を記載する
- 実際に確認できたページは「確認済み」、推定のページは「推定」と備考に明記する
- 404や非公開ページは含めない
- グローバルナビゲーションをベースに階層を整理する

【入力データ】
[以下のいずれかを貼り付ける]
・URLのみ：https://...
・sitemap.xmlの内容：（コピペ）
・sitemap.htmlの内容：（コピペ）
・ナビゲーションのHTML：（コピペ）
```


## 出力形式の例

### Markdown階層形式
```
サイト名/
├─ ホーム / https://example.com/
├─ 事業紹介 / https://example.com/business/
│  ├─ サービスA / https://example.com/business/service-a/
│  └─ サービスB / https://example.com/business/service-b/
├─ 会社概要 / https://example.com/company/
│  ├─ 会社概要 / https://example.com/company/profile/
│  └─ 沿革 / https://example.com/company/history/
└─ お問い合わせ / https://example.com/contact/
```


### CSV（sitemap.csv）
第1〜3階層を列で分けた構成。列はA〜L（12列）。

```
（空白行）
,,第1階層,,第2階層,,第3階層,,階層,,URL,メモ
（空白行）
,1,TOP,,,,,,/,,https://example.com/,
,,,1,会社について,,,,/company,,https://example.com/company,
,,,,,1,会社概要,,/company/about,,https://example.com/company/about,
（最終行）合計,,合計,,合計,ページ合計,総合計,,,
```


### XLSX（sitemap.xlsx）
- 列構成はsitemap/フォルダーにあるsitema-sample.xlsxとsitemap-sample.csvと同じ（A〜L・12列）
- ヘッダー行・データ行ともに背景色なし
- 枠線なし
- 最終行に階層別ページ数の合計式（SUM関数）
- フォント：Arial統一
