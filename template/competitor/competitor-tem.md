# competitor-tem.md
競合・同業・参考サイトを2軸に配置し、自社が狙うデザイン方向性（ゾーン）を検討・共有するための
ビジュアルデザイン ポジショニングマップを作成するための競合分析ファイル。
成果物：
- `06_デザインチャート/design_positioning_chart.html`（ブラウザ確認用）
- `06_デザインチャート/design_positioning_chart.pptx`（クライアント提案用）
- `06_デザインチャート/thumbs/`（サムネ画像）


## 使い方

### このファイルで何ができるか
競合・同業・参考サイトのデザイントーンを2軸で可視化し、
自社が狙うデザイン方向性（差別化の余白）を1枚の図で関係者と合意するためのファイル。


### 入力として用意するもの

| 用意するもの | 必須／任意 | 説明 |
|---|---|---|
| 競合・参考サイトのURL | 必須 | 競合・同業・参考・自社候補を含む5〜20社程度 |
| 自社の目的・ブランド方針 | 必須 | 「どう見られたいか」「何を差別化したいか」を1〜3行で |
| 2軸のイメージ（ざっくりでOK） | 推奨 | 軸のイメージがあれば伝える。なければClaudeに提案させる |
| 競合サイトのKVスクリーンショット | 任意 | あると精度が上がる。なければClaudeがURLから取得を試みる |
| 既存のポジショニングマップ参考図 | 任意 | 軸のラベルを踏襲したい場合に添付する |


### Claudeへの依頼文（コピーして使う）

**パターンA：URLだけある場合（最小構成）**
```
competitor.mdを参照して、以下の競合サイトのポジショニングマップを作成してください。

自社：[会社名] / URL：https://...
目的：[例：化学メーカーとして「信頼感×専門性」で差別化したい]

競合・参考サイト：
- A社（競合）：https://...
- B社（同業）：https://...
- C社（トーン参考）：https://...

軸のイメージ：[例：縦＝安心感⇔躍動感、横＝親しみ⇔専門志向。なければ提案して]
```

**パターンB：KVスクリーンショットも渡す場合（精度重視）**
```
competitor.mdを参照して、添付のKVスクリーンショットをもとに
ポジショニングマップを作成してください。

自社：[会社名]
目的：[例：リニューアルで「専門性は保ちつつ、親しみやすさ」を加えたい]
軸：縦＝[〇〇⇔〇〇] / 横＝[〇〇⇔〇〇]

添付：各社のKVスクリーンショット（ファイル名＝社名）
```


### 成果物の出力形式

用途に応じて出力形式を選ぶ。依頼文に「出力形式：〇〇」と追記するだけで切り替えられる。

| 形式 | 用途 | 特徴 |
|---|---|---|
| HTML | 社内確認・ブラウザでそのまま見せる | インタラクティブ。サムネ画像が埋め込まれる |
| PPTX | クライアント提案・会議での共有 | PowerPointで開ける。スライドとして編集可能 |
| PNG | メール添付・チャット共有 | HTMLをスクショして書き出し。編集不可だが手軽 |

**依頼文への追記例**
```
出力形式：HTML（ブラウザで確認用）
出力形式：PPTX（クライアント提案用・1スライドに収める）
出力形式：HTML + PPTX両方
```

**PPTXの場合の注意点**
- 1スライドに収めることを基本とする
- サムネ画像はPPTX内に埋め込む
- 軸ラベル・凡例・判断基準・所見をスライド内に含める
- フォントはクライアント環境に合わせて指定する（指定なければ游ゴシック）


### 軸の決め方ガイド

軸はこのファイルで一番重要な設定。以下を参考に決める。

| よくある軸の組み合わせ | 向いている業種・目的 |
|---|---|
| 縦：安心感⇔躍動感 / 横：親しみ⇔専門志向 | BtoB製造業・化学・商社 |
| 縦：高級感⇔カジュアル / 横：伝統⇔革新 | 食品・コスメ・ファッション |
| 縦：信頼⇔エネルギッシュ / 横：シンプル⇔情報量多め | IT・SaaS・スタートアップ |
| 縦：やさしい⇔クール / 横：ローカル⇔グローバル | 医療・教育・地域サービス |

- 軸のイメージがなければ「業種と目的」だけClaudeに伝えれば提案してくれる
- 軸は必ず発注者と合意してから座標を確定させる（ここがズレると図全体がズレる）


## 目的
- 競合・参考サイトのデザインのトーンを俯瞰し、自社サイトが狙う立ち位置（差別化の余白）を1枚で可視化・合意する
- 使いどき：アートディレクション検討の初期段階。ブランドメッセージ確定後〜方向性提案時


## 2軸の定義（Phase 1・最重要）

縦軸と横軸それぞれ「両端の対立概念」を決める。軸の意味は必ず発注者と合意すること。

| 軸 | マイナス方向 | プラス方向 |
|---|---|---|
| 縦軸（Y軸） | [例：躍動感・先進性] | [例：安心感・信頼感] |
| 横軸（X軸） | [例：親しみ・カジュアル] | [例：専門・技術志向] |

- 軸は案件のブランド戦略に合わせて言い換える
- 参考図があれば踏襲し、自社向けにラベルを差し替える


## 掲載サイト一覧（Phase 2）

カテゴリ分け：
- **競合**：直接競合の同業種
- **同業・商社**：自社の直接ピア
- **トーン参考**：デザインの雰囲気だけ参考にする他業種
- **自社候補**：自社のデザイン案（比較目的で外すこともある）

| No | 会社名 | URL | カテゴリ | サムネ取得状況 |
|---|---|---|---|---|
| 1 | [例：A社] | https://example.com | 競合 | 取得済み |
| 2 | [例：B社] | https://example.com | 同業・商社 | 白紙（要対応） |
| 3 | [例：C社] | https://example.com | トーン参考 | 取得済み |
| 4 | [例：D社] | https://example.com | 自社候補 | 取得済み |

※サムネ取得できない場合（SPA等）は「白紙（要対応）」と記載し、`4-3フォールバック`対応を行う


## ポジショニング座標（Phase 4〜5）

各サイトを主観評価で配置する。座標は「主観の仮値」であり、関係者レビューで合意形成するためのもの。
断定しない旨を図に明記すること。

| No | 会社名 | カテゴリ | 縦（Y）座標 | 横（X）座標 | 備考 |
|---|---|---|---|---|---|
| 1 | [例：A社] | 競合 | [例：78] | [例：82] | [例：信頼感・専門寄り] |
| 2 | [例：B社] | 同業・商社 | [例：75] | [例：82] | [例：中央やや上] |
| 3 | [例：C社] | トーン参考 | [例：75] | [例：78] | [例：ほぼ中央] |
| 4 | [例：D社] | 自社候補 | [例：-] | [例：-] | [例：取得不可のため比較不能] |

※座標は0〜100で記載（左=0%〜右=100% / 上=0%〜下=100%）


## 座標の移動量（スクショ検証後・Phase 5）

同じKV画像を複数回AIに判定させた場合のブレを記録する。

| 会社名 | 縦（信頼⇔躍動）移動 | 横（親しみ⇔専門）移動 | 移動量（合計） |
|---|---|---|---|
| [例：A社] | [例：78→46（−32）] | [例：82→52（−30）] | [例：約44（軸の約4割・最大）] |
| [例：B社] | [例：75→62（−13）] | [例：82→56（−26）] | [例：約29] |
| [例：C社] | [例：75→73（−2）] | [例：78→73（−5）] | [例：約5（ほぼ動かず）] |
| [例：D社] | [例：-] | [例：-] | [例：取得不可のため比較不能] |

- ボックスが大きいほど「毎回のブレが大きい＝判断が不安定」
- 縦＝信頼（上）の運動 / 横＝親しみ（左）⇔専門（右）


## 再現性検証（Phase 5・Q3）

同じKV画像を5回・独立に判定した時のブレ幅を記録する。

**結果**
- 構図は毎回同じ＝結論は変わらない
- 点の位置は毎回少しズレる
- どっちとも取れるデザインの会社は大きくブレる

**わかったこと**
- 全体の並びは毎回同じ。位置は少しだけ揺れる
- 「印象があいまいな会社」は大きく揺れるので、そこだけ人の目で確認が必要


## 目標ゾーン

自社が狙う立ち位置を記載する。競合が密集する領域と空いている領域を見て決める。

- 狙うゾーン：[例：右上（安心感×専門志向）エリア]
- 差別化の余白：[例：競合は中央〜右に密集→差別化余白は上寄り]
- ゾーン図：`design_positioning_chart.html` の破線円／楕円で表示


## 判断基準・ドラフト所見

| 観点 | 内容 |
|---|---|
| 目的適性 | URLは文章情報のみのため、スクショでないと把握できない |
| 弁別力 | 競合のような同一業種の場合差をつけられるか |
| 情報取得成功率 | 情報が正しく取得できるか。URLもスクショも一部手作業発生 |

所見：[例：同業は右上に密集→差別化余白は下寄り]

## 留意事項

- これはワイヤーフレームではない：グレースケール化は不要。各サイトの実サムネを使う
- 座標は主観の叩き台。関係者レビューで合意形成するためのもの。断定しない旨を図に明記
- 社外サイトのスクリーンショットの取り扱い：社内検討資料に限定し、公開資料への転載可否は別途確認
- 画像の読み込み対象はKV（ファーストビュー）基準。スクロールで印象が変わるサイトは取りこぼす可能性あり
