# CHANGELOG-tem.md
目的（デザイン／開発／運用）に合わせてCHANGELOG.mdを作成するための構成テンプレート。
スタイルルールは `template-style.md` に従うこと。


## 使い方
- 新しいリポジトリやフォルダーにCHANGELOG.mdを作成する時にこのテンプレートを使う
- プロジェクトの目的（デザイン／開発／運用）に応じて、該当するカテゴリの使い方表を参照して内容を埋める
- 参照：[Keep a Changelog 公式（日本語）](https://keepachangelog.com/ja/1.0.0/)


## 背景・前提条件
- CHANGELOG.mdとは、プロジェクトの各バージョンに対する注目に値する変更点を時系列順に並べたファイル
- **機械のためではなく人間のために**書くことが基本理念
- コミットログの差分をCHANGELOG.mdの代わりに使うのは良くない（ノイズが多く、なぜ変更したかの背景が伝わらないため）

### 公式の変更カテゴリ（6種類）
| カテゴリ | 意味 |
|---|---|
| `Added` | 新機能の追加 |
| `Changed` | 既存機能の変更 |
| `Deprecated` | 間もなく削除される機能（廃止予定） |
| `Removed` | 今回で削除された機能 |
| `Fixed` | バグ修正 |
| `Security` | 脆弱性に関する対応 |

### GitのコミットログとCHANGELOG.mdの違い
| | Gitコミット | CHANGELOG.md |
|---|---|---|
| 目的 | 何を変えたかの細かい記録 | なぜ重要な変更だったかのサマリー |
| 読み手 | 主に開発者自身 | 人間全般（チーム・クライアント・将来の自分） |
| 粒度 | 1コミット＝1変更 | 複数コミットをまとめて記録することもある |

### デザイン｜CHANGELOG.md（デザイン採用理由の記録）
| カテゴリ | デザインでの使い方 | 記録例 |
|---|---|---|
| `Added` | 新しいデザイン要素・コンポーネントを追加した | ヒーローセクションのアニメーションを新規追加 |
| `Changed` | 既存のデザインルール・配色・レイアウトを変更した | メインカラーを`#1A1A1A`から`#2C3E50`に変更（クライアント要望） |
| `Deprecated` | 今後使わなくなる予定のデザイン要素を明記 | カルーセルUIを次フェーズで廃止予定（UX改善のため） |
| `Removed` | デザイン要素・コンポーネントを削除した | サイドバーナビゲーションを削除（モバイル優先設計に変更） |
| `Fixed` | デザインの誤り・コンセプトとのズレを修正した | AI感検出でfloating cardを検出、design-tokens.mdの値に戻した |
| `Security` | ほぼ使わない | — |

### 開発｜CHANGELOG.md（実装判断の記録）
| カテゴリ | 開発での使い方 | 記録例 |
|---|---|---|
| `Added` | 新しい機能・コンポーネント・APIエンドポイントを追加した | お問い合わせフォームのバリデーション機能を追加 |
| `Changed` | 既存の実装・技術選定・ファイル構成を変更した | ルーティングをPages RouterからApp Routerに変更（Next.js 14対応） |
| `Deprecated` | 今後使わなくなる予定の実装を明記 | REST APIを次フェーズでGraphQLに移行予定 |
| `Removed` | 機能・ファイル・依存パッケージを削除した | 未使用のmoment.jsを削除（day.jsに移行済み） |
| `Fixed` | バグ・表示崩れ・APIエラーを修正した | SPでハンバーガーメニューが開かない不具合を修正 |
| `Security` | 脆弱性・認証・権限に関する対応をした | 管理画面へのアクセスにセッション認証を追加 |

### 運用｜CHANGELOG.md（改善判断の記録）
| カテゴリ | サイト分析での使い方 | 記録例 |
|---|---|---|
| `Added` | 新しい計測・施策・コンテンツを追加した | GA4にCVイベントのトラッキングを追加 |
| `Changed` | 既存ページ・CTAの文言・導線を変更した | TOPのCTAボタンを「詳しく見る」から「無料相談する」に変更（CVR改善のため） |
| `Deprecated` | 効果が出なくなった施策を廃止予定と明記 | バナー広告を次月で停止予定（CTR 0.1%以下のため） |
| `Removed` | 効果のないページ・コンテンツ・施策を削除した | 離脱率90%超のキャンペーンページを削除 |
| `Fixed` | 計測ミス・リンク切れ・フォームエラーを修正した | お問い合わせフォームの送信エラーを修正（CV計測が抜けていた） |
| `Security` | ほぼ使わない | — |


## 制約事項・禁止事項
- コミットログの差分をそのままCHANGELOG.mdに転記しない
- 「何を変えたか」だけでなく「なぜその変更・判断をしたか」の理由を必ず記載する
- 該当するカテゴリ（Added/Changed/Deprecated/Removed/Fixed/Security）以外は記載しない
- 最新の変更が先頭にくるようにする（逆時系列）
- 日付は`YYYY-MM-DD`形式（ISO 8601）で記載する
- `Unreleased`セクションを上部に置き、作業中の変更を追跡できるようにする


## 出力形式
このルールに従って以下の構成でCHANGELOG.mdを作成する


**出力の順番**
- `# CHANGELOG.md` ：ファイル名をそのまま記載
- `## [Unreleased]` ：作業中でまだリリースしていない変更をまとめる
- `## [YYYY-MM-DD]` ：リリースした日付ごとにセクションを分ける（新しい日付が上）
- `### Added` `### Changed` `### Deprecated` `### Removed` `### Fixed` `### Security` ：該当するカテゴリのみ記載し、変更内容と理由を箇条書きで記載


**出力の例**
```
# CHANGELOG.md

## [Unreleased]

## [2026-07-22]

### Added
- ○○を追加（理由：〜のため）

### Changed
- ○○を変更（理由：〜のため）

### Fixed
- ○○を修正（理由：〜のため）
```
