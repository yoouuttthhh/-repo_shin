# asana-tem.md
Asanaのタスク概要欄に入れる説明文を、誰が読んでも理解できる形でClaudeに作成してもらうためのテンプレート。
スタイルルールは `template-style.md` に従うこと。


## 使い方

- asana及び`asana-tasks.md` に新しいタスクを追加する時にこのテンプレートを参照する
- タスク概要は**タスクを開始する時に追記**する（最初から全部書かない）
- 追記後、Asana MCPを利用してAsanaとGitHubに同時反映する


## タスクの階層構造

```
# 親タスク
└─ ## サブタスク1（タスクプロセス）
    └─ ### サブタスク2（プロセスのタスク・ここからクリアしていく最小単位）
```

- 親タスクは `#` から始める
- サブタスク1は `##` から始める
- サブタスク2は `###` から始める（なければ、省略する）
- 各タスクの直後に「タスク概要」を記述する


## タスク概要の形式

### 親タスクの概要
```
# タスク名
このプロジェクトが存在する理由・解決する問題を1〜2行で記載

**タスク構造**
（サブタスク1・2の全体像をツリーで表示）

**参照**
- [リポジトリ名](GitHubのURL)

**完了条件**
- 完了条件を記載
```

**親タスクの例**
本タスクは、サイトがGoogleの検索エンジンに正しくインデックス（評価・表示）される状態を維持するための、SEOチェック用タスクテンプレートです。

正しくSEO対策が行われていない場合、サイトの検索順位や集客力が低下し、ビジネスチャンスを大きく逃すリスクがあります。サイトのリニューアルや定期アップデートのタイミングで、必ずこのチェックリストを実行してください。


タスク運用のルール（ステータス管理）
下記のサブタスク（手順）を上から順番に確認・実行してください。
各サブタスク内には、具体的に「何をやるか（やり方）」と「どうやって確認するか（確認方法）」が記載されています。

**開始する場合**：サブタスクをどれか1つでも開始した時点で、この親タスク自体を「進行中」セクションに移動させてください。
**一時中断・保留する場合**：一部のサブタスクのみを完了させ、残りの作業を一旦保留（ストップ）する場合は、親タスクを「ToDo」に戻してください。
**完了の処理**：すべてのサブタスクが完了したら、この親タスクを「Done」に移動してください。

```
## タスク名

**含まれるタスク**
- サブタスク2の名前を列挙
```

### サブタスク2（`###`）の概要
```
### タスク名
このタスクが存在する理由・解決する問題を記載

**用語**
- 用語名：一言説明（専門用語がある場合のみ記載）

**完了条件**
- 完了条件を記載

**参照**
- `参照先ファイルパス` または [リンク名](URL)
```


## Claudeへの指示文（タスク概要を追記する時に使う）

```
asana-tem.mdのフォーマットに従って、以下のタスクの概要を作成してください。

タスク名：[タスク名を入力]
階層：[親タスク / サブタスク1 / サブタスク2]
目的：[このタスクの目的を入力]
用語：[専門用語があれば入力。なければ「なし」]
完了条件：[完了の基準を入力]
参照先：[関連ファイルのパスやURL。なければ「なし」]

作成後、asana-tasks.mdの該当箇所に追記し、
Asana MCPを使ってAsanaにも同じ内容を反映してください。
```
