# asana-tem.md
Asanaのタスク概要欄に入れる説明文を、誰が読んでも理解できる形でClaudeに作成してもらうためのテンプレート。
スタイルルールは `template-style.md` に従うこと。


## 使い方
- asana及び`asana-tasks.md` に新しいタスクを追加する時にこのテンプレートを参照する
- タスク下層は基本「親タスク」と「サブタスク」のみとする
- タスク概要は**タスクを開始する時に追記**する（最初から全部書かない）
- 追記後、Asana MCPを利用してGitHubの`asana-tasks.md` をAsanaに同期させる


**タスクの基本階層構造**
```
# 親タスク
└─ ## サブタスク1（タスクプロセス）
    └─ ### サブタスク2（プロセスのタスク・ここからクリアしていく最小単位）
```

- 親タスクは `#` から始める
- サブタスクは `##` から始める
- 各タスクの直後に「タスク概要」を記述する


## 制約事項・禁止事項
特になし

## 出力形式
このルールに従って以下の構成でAsanaとGitHubに同時反映する


### 親タスク


**出力の順番**
- `# 親タスク` ：タスク名をそのまま記載
- 何のためのプロジェクト及びタスクか・解決する課題を1〜3行で記載
- `**使い方**` ：該当するタスクの使い方・更新ルールを箇条書きで記載
- `**タスク階層構造**` ：親タスクからサブタスク2をWBS形式のツリーで記載
- `**関連リンク**` ：関連するサイト・リポジトリ・ドキュメントのリンクを記載
- `**注意点**` ：気をつけないといけないことがある場合、留意事項を記載（任意）

正しくSEO対策が行われていない場合、サイトの検索順位や集客力が低下し、ビジネスチャンスを大きく逃すリスクがあります。
サイトのリニューアルや定期アップデートのタイミングで、必ずこのチェックリストを実行してください。

- **開始する場合**：サブタスクをどれか1つでも開始した時点で、この親タスク自体を「進行中」セクションに移動させてください。

### サブタスク


**出力の順番**
- `## サブタスク` ：タスク名をそのまま記載
- 何のためのプロジェクト及びタスクか・解決する課題を1〜3行で記載
- `**使い方**` ：該当するタスクの使い方・更新ルールを箇条書きで記載
- `**作業手順**` ：該当するタスクの作業フローを1〜3行で記載するか、ステップで記載（指示がなければ、基本なしとする）
- `**確認手順**` ：この作業の確認者に対して、確認する方法を1〜3行で記載（指示がなければ、基本なしとする）
- `**完了条件**` ：どの状態であればタスク完了なのか箇条書きで記載（指示がなければ、基本なしとする）
- `**関連リンク**` ：関連するサイト・リポジトリ・ドキュメントのリンクを記載
- `**関連用語**` ：専門的な言葉の意味をシンプルに説明し、理解を助ける（指示がなければ、基本なしとする）
- `**注意点**` ：気をつけないといけないことがある場合、留意事項を記載
