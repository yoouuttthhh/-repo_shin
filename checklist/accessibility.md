# ウェブアクセシビリティ チェックリスト（WCAG準拠・サイト公開前チェック用）

> サイトを公開・更新するたびにこのチェックリストを確認してください。
> 目安レベル：A（必須）／AA（推奨・多くの公的基準の目標水準）

## 1. 知覚可能（Perceivable）

### 1.1 テキスト以外のコンテンツ
- [ ] 画像・アイコン・グラフに意味を伝える代替テキスト（alt）を設定した
- [ ] 装飾目的の画像は空のalt（alt=""）にするか読み上げ対象から除外した
- [ ] アイコンのみのボタン・リンクにアクセシブルな名前（aria-labelなど）を設定した

### 1.2 動画・音声コンテンツ
- [ ] 音声のみ／映像のみのコンテンツにテキスト版の代替を用意した
- [ ] 収録済み動画に字幕（キャプション）を付けた
- [ ] ライブ配信にも字幕を付けた（AA以上）
- [ ] 映像の内容を音声解説または代替テキストで補足した

### 1.3 構造と関係性
- [ ] 見出し・リスト・表を意味に応じたHTML要素（h1-h6, ul/ol, table等）でマークアップした
- [ ] 画面上の視覚的な順序と、支援技術が読み上げる順序が一致している
- [ ] 「右上のボタン」「赤い項目」など形・色・位置だけに頼った説明をしていない

### 1.4 見やすさ・聞きやすさ
- [ ] 色だけで意味の違い（エラー・必須項目など）を伝えていない
- [ ] 自動再生される音声を停止・一時停止する手段がある
- [ ] 通常テキストのコントラスト比が4.5:1以上、大きな文字は3:1以上ある
- [ ] テキストを200%拡大しても内容やレイアウトが破綻しない
- [ ] 文字情報を画像化せず、実テキストとして表示している（ロゴ等を除く）

## 2. 操作可能（Operable）

### 2.1 キーボード操作
- [ ] マウスを使わずキーボードだけで全ての機能を操作できる
- [ ] キーボードフォーカスが特定の要素から抜け出せなくなる箇所がない

### 2.2 十分な時間
- [ ] 制限時間がある操作は、延長・解除・調整ができる
- [ ] 自動スクロール・カルーセル・自動更新を一時停止できる

### 2.3 発作・身体的反応への配慮
- [ ] 1秒間に3回を超える激しい点滅・フラッシュがない

### 2.4 ナビゲーション
- [ ] 各ページのタイトルが内容を的確に表している
- [ ] 見出しやランドマーク（header/nav/main等）でページ構造を示している
- [ ] リンクの文言だけでリンク先の内容が推測できる（「こちら」等を避ける）
- [ ] タブキーでのフォーカス移動順序が論理的である
- [ ] 現在フォーカスされている要素が視覚的に分かる

## 3. 理解可能（Understandable）

### 3.1 読みやすさ
- [ ] HTMLのlang属性でページの言語を指定している
- [ ] 専門用語・略語・難読語に説明や読み仮名を用意している

### 3.2 予測可能な動作
- [ ] フォーム入力や要素へのフォーカスだけで予期しない画面遷移が起きない
- [ ] ナビゲーションメニューなどがページ間で一貫している

### 3.3 入力支援
- [ ] フォームの各項目に分かりやすいラベルが付いている
- [ ] 入力エラー発生時に、原因と修正方法を具体的に提示している
- [ ] 送信・削除など重要な操作の前に内容確認・訂正ができる

## 4. 堅牢性（Robust）

### 4.1 互換性
- [ ] HTMLの構文に誤りがない（タグの閉じ忘れ、id重複などがない）
- [ ] 独自UIコンポーネント（モーダル、タブ、アコーディオン等）にARIAロール・状態・プロパティを適切に設定している

## 運用メモ
- 公開前に本チェックリストを一通り確認する
- axe DevTools／Lighthouse／WAVEなどの自動チェックツールも併用する（自動検出できない項目も多いため目視確認は必須）
- プロジェクトの要件に応じて対象レベル（A／AA／AAA）を決め、必要な項目のみ運用してもよい
