# dictionary.md
人間もClaudeも読み取りやすい形式で管理するコマンド集。
新しいコマンドを追加する際は `wiki-template.md` と同じ要領でClaudeに依頼する。


## ターミナル・VS Code

| やること | コマンド | 説明 |
|---|---|---|
| フォルダーに移動する | `cd /path/to/your-repo` | 指定したフォルダーに移動する |
| VS Codeを現在フォルダーに切り替える | `code -r .` | 今開いているウィンドウをこのフォルダーに切り替える |
| VS Codeを新しいウィンドウで開く | `code .` | 新しいウィンドウでVS Codeを開く |


## Git

| やること | コマンド | 説明 |
|---|---|---|
| 変更状況を確認する | `git status` | 変更されたファイルの一覧と状態を確認する |
| 全ファイルをステージに上げる | `git add .` | 変更されたファイルをすべてステージに追加する |
| 特定ファイルをステージに上げる | `git add <ファイル名>` | 指定したファイルだけステージに追加する |
| コミットする | `git commit -m "説明"` | ステージの変更を履歴として確定する |
| プッシュする | `git push` | ローカルの変更をGitHubに送信する |
| 初回プッシュする | `git push -u origin <ブランチ名>` | ブランチをGitHubに登録しながらプッシュする |
| GitHubから取得する | `git pull` | GitHubの最新状態をローカルに取り込む |
| リモート設定を確認する | `git remote -v` | どのGitHubリポジトリと繋がっているかを確認する |
| ブランチ一覧を確認する | `git branch` | ローカルのブランチを全て表示する（現在のブランチに * が付く） |
| ブランチを切り替える | `git switch <ブランチ名>` | 既存のブランチに切り替える |
| 新しいブランチを作って切り替える | `git switch -c <新しいブランチ名>` | 新しいブランチを作成して切り替える |


## Node.js

| やること | コマンド | 説明 |
|---|---|---|
| バージョン一覧を確認する | `nvm ls` | インストール済みのNode.jsバージョンを表示する |
| 最新LTSをインストールする | `nvm install --lts` | 最新の安定版Node.jsをインストールする |
| 特定バージョンをインストールする | `nvm install 20` | 指定したバージョンのNode.jsをインストールする |
| バージョンを切り替える | `nvm use 20` | 使用するNode.jsのバージョンを切り替える |


## Claude Code

| やること | コマンド | 説明 |
|---|---|---|
| 起動する | `claude` | 現在のフォルダーでClaude Codeを起動する |
| 使えるコマンドを一覧表示する | `/help` | 使えるコマンドを一覧表示する |
| CLAUDE.mdを自動生成する | `/init` | コードベースを解析してCLAUDE.mdを自動生成する |
| 会話履歴をリセットする | `/clear` | 会話履歴をリセットする（CLAUDE.mdは残る）。`/reset` `/new`でも同じ |
| 会話を要約する | `/compact` | 会話を要約してコンテキストを節約する。フォーカス指示も追加できる |
| 履歴に残さず質問する | `/btw` | 会話履歴に残さずサッと質問する |
| チェックポイントに巻き戻す | `/rewind` | 会話・コードを以前のチェックポイントに巻き戻す |
| 使用モデルを切り替える | `/model` | 使用モデルを確認・切り替える |
| トークン使用量を確認する | `/cost` | 今のセッションのトークン使用量と費用を確認する |
| 使用制限を確認する | `/usage` | プランの使用制限と現在のレート制限状況を確認する |
| コンテキストを可視化する | `/context` | コンテキストウィンドウに何が読み込まれているか可視化する |
| CLAUDE.mdを編集する | `/memory` | CLAUDE.mdを確認・編集する |
| フォルダーアクセスを許可する | `/add-dir` | このセッションで追加のフォルダへのアクセスを許可する |
| ツール承認設定を変更する | `/permissions` | どのツールに承認が必要かを確認・変更する |
| 設定を変更する | `/config` | テーマ・デフォルト設定・エディタモード等を変更する。`/settings`でも同じ |
| プランモードに入る | `/plan` | プランモード（読み取り専用）に入る。タスク説明も一緒に入力できる |
| 差分を確認する | `/diff` | 未コミットの変更と各ターンの差分を確認する |
| 回答をコピーする | `/copy` | 最後の回答（またはコードブロック）をクリップボードにコピーする |
| 会話を保存する | `/export` | 現在の会話をファイルまたはクリップボードに保存する |
| MCP接続を管理する | `/mcp` | MCPサーバーの接続・認証を管理する |
| サブエージェントを管理する | `/agents` | サブエージェントの一覧・作成・編集 |
| フック設定を確認する | `/hooks` | ツールイベントのフック設定を確認する |
| スキル一覧を表示する | `/skills` | このセッションで使えるスキルを一覧表示する |
| コードをレビュー・改善する | `/simplify` | 最近変更したファイルを品質・再利用性の観点でレビューして改善を適用する |
| 状態を確認する | `/status` | アカウント・モデル・作業ディレクトリ・バージョンを確認する |
| 環境を診断する | `/doctor` | インストール・環境の問題を診断する |
| 問題を報告する | `/feedback` | セッションのコンテキストを添付してAnthropicに問題を報告する。`/bug`でも同じ |
| セッションを再開する | `/resume` | 以前のセッションを再開する。`/continue`でも同じ |
| 認証を切り替える | `/login` / `/logout` | 認証・アカウント切り替え・サインアウト |
| 終了する | `/exit` | Claude Codeを終了する。`/quit`でも同じ |
