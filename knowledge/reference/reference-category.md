## サイト分析に活用する際の視点
- ターゲット層（年齢・性別・業種）とテイストの整合性
- 配色（明度・彩度・色数）のテイストらしさ
- フォント選定（和文/英字の組み合わせ、ウェイト、字間）
- レイアウト・装飾要素（イラスト、図形、テクスチャ、写真加工）がテイストに合っているか
- キャッチコピーの見せ方（手書き風／筆記体／明朝など）と可読性のバランス

## デザインの印象
### かわいい
- **想定ターゲット**: コスメ・ファッションEC、子供服ブランド、スイーツ系など若年女性・子供向け
- **配色傾向**: 明度の高いパステルカラー中心。文字は可読性のため中明度に調整
- **フォント傾向**: 丸ゴシック系＋サンセリフの英字。やや細めのウェイトと広めの字間で柔らかさを演出。使用フォント種類は絞る
- **制作ポイント**: 小さめのイラスト・写真を散りばめて軽やかさを出す／背景はパステルの単色または淡いバイカラー

### ポップ・カジュアル
- **想定ターゲット**: プレゼントキャンペーン等、明るく賑やかな訴求
- **配色傾向**: 高彩度でカラフル、コントラストを意識
- **フォント傾向**: スリムでやや太めの英字ゴシック、和洋の組み合わせで新鮮さを演出
- **制作ポイント**: 集中線・吹き出し・図形装飾で躍動感を出す／立体表現は明るいカラーの影を使用

### ロマンチック・清楚
- **想定ターゲット**: ブライダル、美容サロン、化粧水キャンペーンなど
- **配色傾向**: パステルにグレーを混ぜた大人っぽいペールカラー
- **フォント傾向**: カリグラフィ系英字＋明朝/セリフ系の和文（太さは中庸）
- **制作ポイント**: 白基調でゴージャスになりすぎないよう抑えた配色に／筆記体は小さく添え、和文で補足し可読性を確保

### ナチュラル
- **想定ターゲット**: 生活雑貨、リラックス系サービスなど
- **配色傾向**: 草木をイメージしたボタニカル配色。暖色/寒色で季節感を演出
- **フォント傾向**: 太めの筆記体をポイント使い、丸ゴシックで読みやすく／高年齢層向けは明朝で高級感を追加
- **制作ポイント**: 暖色でぼやけないよう差し色でキャッチコピーに視線誘導

### クリア・さわやか
- **想定ターゲット**: 清涼飲料水、新卒採用サイトなど
- **配色傾向**: 高明度・高彩度のブルー中心、色数を絞り差し色は暖色1色程度
- **フォント傾向**: 丸みのあるゴシック英字（字間広め）、手書き風フォントで親近感を演出
- **制作ポイント**: グラデーションや透過で透明感を出す／手書きコピーは短く簡潔に

### クール
- **想定ターゲット**: 夏季飲料・スキンケア、かっこよさ訴求全般
- **配色傾向**: イエローで元気さ、ブルーのコントラストで明快さを演出
- **フォント傾向**: 現代的なゴシック英字、ウェイト豊富で可読性の高い和文フォント
- **制作ポイント**: 水しぶき・水滴などの視覚要素、商品特性を表すアイコン制作

### ゴージャス
- **想定ターゲット**: 高級ホテル、受賞・プレミアム訴求など
- **配色傾向**: ゴールド＋黒＋赤、またはパープルで高級感・ラグジュアリー感
- **フォント傾向**: 和文との相性が良いセリフ英字、上品な明朝体
- **制作ポイント**: ゴールドグラデーションは失敗しやすいため要練習／フレーム装飾や質感テクスチャで豪華さを補強

### ワイルド
- **想定ターゲット**: スポーツ・車・酒など
- **配色傾向**: 土や草木を思わせる濁った低明度カラーで力強さを表現
- **フォント傾向**: 数字の可読性が高いフォント、荒々しさを感じさせる明朝体
- **制作ポイント**: 写真は明度を抑えコントラスト強調／重要情報の視認性を優先したレイアウト

### クラシック
- **想定ターゲット**: 伝統・格式を感じさせる商材
- **配色傾向**: ダークトーンで温かみのある配色、アクセントも低明度
- **フォント傾向**: セリフ系の英字で品格を演出、レトロな手書き和文フォント
- **制作ポイント**: ポップになりすぎない配色管理／テクスチャで質感を出しつつ装飾は控えめに

### フォーマル
- **想定ターゲット**: 不動産・金融・フォーマルウェアなど
- **配色傾向**: 黒と青系で誠実・公式な印象
- **フォント傾向**: 個性のあるセリフ英字、格式を感じる明朝体和文
- **制作ポイント**: 黒・青中心の落ち着いた配色／差し色はワンポイントに留める

### シック
- **想定ターゲット**: 大人向けファッション、上品な化粧品など
- **配色傾向**: 薄紫＋青、またはモノトーンでベースとアクセントを近い色でまとめる
- **フォント傾向**: 可読性の高いサンセリフ英字、上品な行書系和文
- **制作ポイント**: 同系色でまとめ派手さを抑える／シンプルながら流行を意識したタイポグラフィ

### モダン
- **想定ターゲット**: 若者向けファッション、トレンド雑貨など
- **配色傾向**: トレンドカラー＋大胆なコントラスト、青緑系はくすみを加える
- **フォント傾向**: 細身でスタイリッシュな英字（雑誌ロゴ的）、新しい和文フォントの活用
- **制作ポイント**: シンプルな斜線構成＋インパクトのある英字／現代アート風の図形構成も有効

## レスポンシブの変化点
- Desktop → Tablet：3〜4カラムのカードが2カラムに変化
- Tablet → SP：ハンバーガーメニューに切替、ヒーロー見出しが2行→3行に折返し
- 気になった点：SPでアイソメトリックイラストが縮小され、
  シグネチャー要素（タイル状の床）の視認性が下がっている

- キャッチコピーの型：具体的な課題提起型（「マンネリ化」「やりっぱなし」等の
  固有の課題ワードを使用、抽象語ではない）
- 文体：ですます調中心、体言止めも一部使用
- 特徴的な言い回し：「◯◯を、まるごと解決」型の動詞締め

**全体印象**
青系グラデーション＋アイソメトリックイラストを基調にした、BtoB SaaSの

## 4. GitHubリポジトリ作成〜サイト立ち上げ手順

### Step 1：リポジトリを作成する
1. GitHubで新規リポジトリ `design-scrapbook` を作成（Public/Privateはお好みで）
2. ローカルにクローン
```bash
git clone https://github.com/【あなたのアカウント名】/design-scrapbook.git
cd design-scrapbook
```

### Step 2：フォルダ構成を作る
```bash
mkdir entries assets site
touch README.md taxonomy.md taste-profile.md disagreements.md
```
上記2・3の内容を、それぞれ `taxonomy.md` と `entries/2026-07-08_kagami.md` として保存します。

### Step 3：初回コミット
```bash
git add .
git commit -m "初期構成: design-scrapbookのフォルダとサンプルエントリを追加"
git push origin main
```

### Step 4：Git LFSの設定（画像・gif容量対策）
スクショやgifは容量が大きくなりやすいため、Git LFSを使います。
```bash
git lfs install
git lfs track "assets/*.png"
git lfs track "assets/*.gif"
git add .gitattributes
git commit -m "Git LFSでassetsを管理"
git push
```

### Step 5：サイト（Next.js）を立ち上げる
```bash
cd site
npx create-next-app@latest . --typescript --app
npm install gray-matter
```
`gray-matter` はmdファイルのfrontmatterを解析するために使用します。

### Step 6：entries/を読み込むロジックを作る
`site/lib/entries.ts` に、`../entries/*.md` をすべて読み込み、
frontmatterと本文をパースして返す関数を実装します（Claude Codeに
「entries/ディレクトリの.mdファイルをすべて読み込み、frontmatterと
本文を返すNext.js用の関数を作って」と依頼すれば生成できます）。

### Step 7：ギャラリーページ・詳細ページを実装
- `app/page.tsx`：entries一覧をカードグリッドで表示、タグでフィルタ
- `app/entry/[slug]/page.tsx`：個別エントリの詳細表示
- `app/taste-profile/page.tsx`：taste-profile.mdをそのまま表示

これも前回のモックアップ（カードグリッド＋詳細ページ）をClaude Codeに
見せながら「このレイアウトで実装して」と依頼すれば、Figma MCPを介さずとも
直接コード化できます。

### Step 8：ローカル確認
```bash
npm run dev
```
`http://localhost:3000` でギャラリーが表示されることを確認します。

### Step 9：公開（Vercelが最も手軽）
1. [vercel.com](https://vercel.com) でGitHubアカウント連携
2. `design-scrapbook` リポジトリをインポート（Root Directoryを `site` に指定）
3. デプロイを実行 → 自動的に公開URLが発行される

### Step 10：以後の運用フロー
```bash
# 新しいスクラップを追加するたび
git add entries/新規ファイル.md assets/新規画像
git commit -m "追加: ○○社サイトのスクラップ"
git push
```
Vercelと連携していれば、pushするだけで自動的にサイトが再ビルドされ、
新しいエントリがギャラリーに反映されます。
