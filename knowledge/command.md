# command.md — コマンドリファレンス

人間もClaudeも読み取りやすい形式で管理するコマンド集。
新しいコマンドを追加する際は `wiki-template.md` と同じ要領でClaudeに依頼する。

## フォーマット

各コマンドは以下の構造で記載する。

```
### コマンド名（動詞+目的語）
# 何をするコマンドか（1行）
コマンド本体
```

## ナビゲーション

### フォルダーに移動する
```zsh
# 指定したリポジトリのフォルダーに移動する
cd /path/to/your-repo
```

### VS Codeを現在のフォルダーに切り替える
```zsh
# 今開いているVS Codeのウィンドウを、このフォルダーに切り替える（-r オプション）
code -r .
```

### VS Codeで現在のフォルダーを新しいウィンドウで開く
```zsh
# 新しいウィンドウでVS Codeを開く
code .
```

## Git：ブランチ操作

### ローカルブランチの一覧を確認する
```zsh
# ローカルにあるブランチを全て表示する（現在のブランチに * が付く）
git branch
```

### ブランチを切り替える
```zsh
# 既存のブランチに切り替える（旧コマンド）
git checkout <ブランチ名>

# 既存のブランチに切り替える（新コマンド・推奨）
git switch <ブランチ名>
```

### 新しいブランチを作って切り替える
```zsh
# 新しいブランチを作成して、そのブランチに切り替える（旧コマンド）
git checkout -b <新しいブランチ名>

# 新しいブランチを作成して、そのブランチに切り替える（新コマンド・推奨）
git switch -c <新しいブランチ名>
```

## Git：変更の記録

### 変更状況を確認する
```zsh
# 変更されたファイルの一覧と状態（add済みかどうか）を確認する
git status
```

### 変更をステージに上げる
```zsh
# 変更されたファイルをすべてステージに追加する
git add .

# 特定のファイルだけステージに追加する
git add <ファイル名>
```

### コミットする
```zsh
# ステージに上げた変更を履歴として確定する
git commit -m "変更内容の説明"
```

### GitHubにプッシュする
```zsh
# ローカルの変更をGitHubに送信する
git push

# 初回プッシュ（ブランチをGitHubに登録しながらプッシュ）
git push -u origin <ブランチ名>
```

## Git：リモート・同期

### リモートの変更を取得する
```zsh
# GitHubの最新状態をローカルに取り込む
git pull
```

### リモートの設定を確認する
```zsh
# どのGitHubリポジトリと繋がっているかを確認する
git remote -v
```

## nvm：Node.jsバージョン管理

### インストール済みのNode.jsバージョンを確認する
```zsh
# インストール済みのNode.jsバージョン一覧を表示する
nvm ls
```

### Node.jsをインストールする
```zsh
# 最新のLTS（安定版）をインストールする
nvm install --lts

# 特定のバージョンをインストールする
nvm install 20
```

### 使用するNode.jsを切り替える
```zsh
# 指定したバージョンに切り替える
nvm use 20
```
