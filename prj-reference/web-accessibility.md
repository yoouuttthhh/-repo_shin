# web-accessibility.md
身体・視覚・聴覚・認知等に障害のある利用者を含む、より多くの人が問題なく利用できる状態にすることを目的とします。
対応にあたっては、国際的なアクセシビリティ規格である WCAG（Web Content Accessibility Guidelines）を判断基準とします。


## 対象範囲
- 対象ページ／機能：（例：全ページ／トップページとお問い合わせフォーム／会員登録フロー など、ここに記入）
- 対象デバイス：PC／スマートフォン／タブレット（該当するものに記入）
- 対象ブラウザ・支援技術：主要ブラウザ最新版、スクリーンリーダー（VoiceOver, NVDA等）での動作を想定


## 遵守基準（達成レベル）
- 基準：WCAG 2.0（または2.1/2.2）
- 目標レベル：シングルA全項目 ＋ ダブルAA全項目への準拠
  - レベルAは必須（未達の場合は重大な利用阻害とみなします）
  - レベルAAは原則対応（対応が困難な項目は理由を報告書に明記）
  - レベルAAAは任意（対応できる範囲で可）


## 作業内容
- 現状のページに対し、レベルA/AAの各達成基準に照らした診断を実施する
- 問題箇所を洗い出し、一覧化する（ページ・箇所・該当基準・重要度を明記）
- 問題箇所を修正する、または修正方針を提示し合意の上で対応する
- 修正後、再検証を行い、問題が解消していることを確認する


## 検証方法
- 自動チェックツール（axe DevTools、Lighthouse、WAVE等）による機械的検証
- キーボードのみでの操作確認（Tabキーでの全機能操作、フォーカス可視性）
- スクリーンリーダーでの読み上げ確認
- カラーコントラスト測定ツールでの色比率確認
- 自動ツールで検出できない項目（意味的な構造、代替テキストの適切さ等）は目視・実機で確認する
